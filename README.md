# 事件驱动选股器

基于新闻舆情和公告事件的产业趋势选股策略。

## 核心逻辑

产业趋势（订单/技术突破）+ 业绩拐点 + 资金确认

### 三层过滤

1. **产业趋势**（权重40%）
   - 订单/合同公告
   - 技术突破新闻
   - 产能扩张
   - 涨价/供不应求
   - 政策支持

2. **业绩拐点**（权重30%）
   - 营收增速 > 30%
   - 容忍净利润亏损（早期成长股）

3. **资金确认**（权重30%）
   - 主力资金流入为正
   - 近30天相关新闻 >= 3条

## 事件关键词

| 类型 | 关键词 |
|-----|--------|
| 订单 | 订单、合同、中标、签订、大单、采购、供货、合作 |
| 技术突破 | 量产、突破、国产化、替代进口、技术领先、专利 |
| 产能扩张 | 扩产、投产、新建、产能、产线、工厂 |
| 涨价 | 涨价、供不应求、库存下降、紧缺、价格上调 |
| 政策支持 | 国家战略、专项资金、扶持政策、产业规划 |

## 数据源

| 接口 | 来源 | 数据 |
|-----|------|------|
| stock_yjbb_em | 东财 | 业绩报表（营收增速）|
| stock_fund_flow_individual | 同花顺 | 个股资金流 |
| stock_news_em | 东财 | 个股新闻 |

## 运行

```bash
python3 event_driven_screener.py
```

## 筛选条件

| 条件 | 值 | 说明 |
|-----|---|------|
| 营收增速 | > 30% | 业绩拐点 |
| 主力资金 | > 0 | 资金认可 |
| 新闻数量 | >= 3条 | 近30天事件信号 |
| 综合得分 | >= 60分 | 及格线 |

## 评分规则

| 维度 | 分数 | 条件 |
|-----|------|------|
| 营收增速 | 10-20分 | >30%=10分, >50%=15分, >100%=20分 |
| 订单事件 | 15分 | 有订单/合同新闻 |
| 技术突破 | 15分 | 有技术突破新闻 |
| 产能扩张 | 10分 | 有扩产新闻 |
| 资金流向 | 10-20分 | >2万=15分, >5万=20分 |

## 与业绩选股器的区别

| 维度 | 业绩选股器 | 事件驱动选股器 |
|-----|-----------|--------------|
| 核心 | 业绩高增长 | 产业趋势事件 |
| 净利润 | 必须>100% | 容忍亏损 |
| 数据 | 财务报表 | 新闻+公告 |
| 时效 | 滞后 | 实时 |
| 目标 | 业绩爆发股 | 产业趋势龙头股 |

## 适用场景

- 找0→1产业突破的早期机会
- 抓订单/技术突破的启动点
- 容忍短期亏损，看长期趋势
